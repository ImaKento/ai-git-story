# gi-git-report 新コマンド設計

`gi-git-report` の CLI を、**「何をしたいか」= コマンド**、**「いつの分か」= 共通フラグ**に整理して、直感的で迷わないインターフェースにする提案です。

## 目的

- 操作の一貫性を高める
- 学習コストを下げる
- 初心者でも迷わず使える導線を用意する

## 1. インタラクティブ・モード

`ai-git` 単体では何も実行しません。`ai-git report` を実行すると、メニューから目的を選べます。

```bash
ai-git report
```

表示イメージ:

```text
? どのレポートを生成しますか？ (Use arrow keys)
❯ 📖 今日のまとめ (Today)
  🗓️ 今週の振り返り (This Week)
  🔍 期間を指定する (Custom Range)
  📊 チームの貢献度 (Team Stats)
  ⚙️ 設定を確認する (Config)
```

## 2. コマンド体系

主要な機能は、以下の 2 つに集約します。

- `report`: 開発レポート（日記）を生成
- `stats`: 貢献度や活動の統計を分析

### 📖 開発レポート生成 (`report`)

コミット履歴を日記形式のレポートとして出力します。

- 今日のレポート

```bash
ai-git report -t
```

- 今週のレポート

```bash
ai-git report -w
```

- 期間指定のレポート

```bash
ai-git report -s "3 days ago"
```

### 📊 貢献度の分析 (`stats`)

貢献度ランキングや活動分析を出力します。

- 今日の統計

```bash
ai-git stats -t
```

- 今週の統計

```bash
ai-git stats -w
```

- 期間指定の統計

```bash
ai-git stats -s "2024-04-01"
```

## 3. 共通オプション

すべてのコマンドで同じショートカットを使えます。

| オプション | 短縮形 | 説明 |
| --- | --- | --- |
| `--today` | `-t` | 「今日」を対象にする |
| `--week` | `-w` | 「過去 7 日間」を対象にする |
| `--since` | `-s` | 開始日を指定（例: `"2024-01-01"`, `"2 days ago"`） |
| `--until` | `-n` | 終了日を指定 |
| `--user` | `-u` | ユーザーを絞り込む（引数なしで選択リストを表示） |
| `--lang` | `-l` | 言語切り替え（`ja` / `en`） |

## 改善ポイント

### 一貫性の確保

`report` でも `stats` でも、`-t` や `-w` など同じフラグで期間を切り替えられるため、操作に迷いません。

### 学習コストの低減

基本コマンドは `report` と `stats` の 2 つだけです。あとは共通オプションを組み合わせるだけの「積み木方式」で使えます。

### 初心者にやさしい導線

コマンドを忘れても `ai-git report` でインタラクティブメニューを開き、機能へたどれます。
